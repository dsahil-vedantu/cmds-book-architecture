SIMILARITY LEVELS — the active regeneration modes
═══════════════════════════════════════════

Each similarity level defines what is LOCKED (must stay the same) and what CHANGES. The question_type tells you the output format.

───────────────────────────────────────────
1. NUMBERS ONLY (Most Conservative — exact clone with new values)
───────────────────────────────────────────
   LOCKED  : chapter, topic, concept, solution method, answer logic, AND the
             exact wording, sentence structure, scenario, objects, names, and
             context of the source — everything except the numbers
   CHANGES : ONLY the numeric values in the stem (and the recomputed answer)

   What to do:
   - Change every numeric value in the stem to new, clean, curriculum-appropriate numbers.
   - Keep the wording VERBATIM otherwise — do NOT rephrase, restructure, or swap
     scenarios/objects/names. The ONLY difference from the source is the numbers.
   - Recompute the answer and solution for the new numbers using the SAME method.
   - Pick numbers that yield clean, exam-appropriate answers (no ugly arithmetic).
   - Because the numbers changed, any figure/diagram tied to those numbers may need
     updating to match (handled separately by the figure-regen step).

   Question type: ALWAYS keep the SAME type as the source for this mode, unless
   question_type is explicitly specified.

───────────────────────────────────────────
2. NUMBERS AND REPHRASE (First Choice / Default)
───────────────────────────────────────────
   LOCKED  : chapter, topic, concept, solution method, answer logic
   CHANGES : all numeric values AND the wording/phrasing

   What to do:
   - Change ALL numeric values in the stem.
   - Completely rephrase the stem — different sentence structure, not just synonym swaps.
   - Change scenario details: use different objects, names, locations, and physical contexts.
   - The underlying concept and solution approach must remain identical.
   - For competitive exam contexts (JEE Main, JEE Advanced, NEET):
     * Maintain rigorous physical/mathematical scenarios and multi-step reasoning.
     * Ensure the numbers chosen yield precise, clean answers without arithmetic inconsistencies.
     * Generate distractors that reflect common exam slips (e.g., wrong sign conventions, omitting a factor of 2, small algebraic errors, or dimensional inconsistencies).
   - The regenerated version must feel like it was written by a different question author.

   Question type: Keep same as source UNLESS question_type is explicitly specified.

───────────────────────────────────────────
3. NUMBERS, REPHRASE AND ADD CONCEPT (Second Choice)
───────────────────────────────────────────
   LOCKED  : chapter, topic, original concept
   CHANGES : wording, phrasing, all numbers, AND layers exactly ONE additional concept from the same topic on top

   What to do:
   - Apply the full rephrasing and numerical changes of "numbers_and_rephrase".
   - Start with the source's core concept, then integrate exactly ONE more concept from the same topic that the student must apply to solve the problem.
   - The question now requires mastery of BOTH concepts to solve, naturally increasing the academic depth.
   - The added concept must be topic-appropriate and curriculum-aligned.

   Question type: Use source type UNLESS question_type is explicitly specified.

───────────────────────────────────────────
4. NEW QUESTION SAME TOPIC (Optional 1)
───────────────────────────────────────────
   LOCKED  : chapter, topic
   CHANGES : everything else — new scenario, new setup, new numbers, new twist, new angle

   What to do:
   - Create a brand-new original question on the same topic.
   - Use a completely different scenario, setup, and context.
   - NOT a rephrase — a genuinely new question.
   - The question should feel like it came from a different question paper.
   - May test the same concept from a different angle or test a different concept within the same topic.

   Question type: Use source type UNLESS question_type is explicitly specified.

───────────────────────────────────────────
5. SAME TOPIC + ADD ONE CONCEPT (Optional 2)
───────────────────────────────────────────
   LOCKED  : chapter, topic, original concept
   CHANGES : layers ONE additional concept from the same topic on top

   What to do:
   - Start with the source's core concept.
   - Add exactly ONE more concept from the same topic that the student must also apply.
   - The question now requires mastery of BOTH concepts to solve.
   - Difficulty naturally rises.
   - The added concept must be topic-appropriate and curriculum-aligned.

   Question type: Use source type UNLESS question_type is explicitly specified.

───────────────────────────────────────────
6. SAME CHAPTER ANY TOPIC (Optional 3)
───────────────────────────────────────────
   LOCKED  : only the chapter
   CHANGES : topic, sub-topic, concept — all free for you to choose

   What to do:
   - Use the source only as inspiration for difficulty level and question style.
   - Pick any topic within the same chapter.
   - Create a completely independent question.
   - Match the source's difficulty tier and cognitive demand.

   Question type: Use source type UNLESS question_type is explicitly specified.

═══════════════════════════════════════════
QUESTION TYPES
═══════════════════════════════════════════
When question_type is specified, the output MUST use that format regardless of the source's format. When question_type is "same_as_source" or blank, match the source.

  Objective_SCQ        — single correct option, 4 choices (A/B/C/D), answer is one letter
  Objective_MCQ        — one or more correct options, 4 choices, answer is a set like {A, C}
  Binary_TF            — True or False with a 1-line justification
  Integer_SingleDigit  — answer is 0–9
  Integer_DoubleDigit  — answer is 10–99
  Integer_NDigits      — answer is any integer
  Numerical_2Decimal   — answer is a decimal rounded to 2 places
  Matching_RadioGrid   — Column A matched to Column B, answer as A→3, B→1, etc.
  Subjective_VeryShort — 1–2 sentence answer
  Subjective_Short     — short paragraph (4–6 sentences)
  Subjective_Long      — full multi-paragraph derivation or explanation
  Comprehension        — shared passage followed by sub-questions
  AssertionReasoning   — Assertion + Reason format
  FillInTheBlanks      — stem with blanks to fill

When CONVERTING question types:
- Preserve the core concept being tested.
- Adapt the stem to fit the new format naturally.
- For MCQ → Subjective: remove options, ask the student to derive/explain.
- For Subjective → MCQ: create stem from the question, generate 4 plausible options with distractors from common errors.

RULES
═══════════════════════════════════════════

RULE 1 — CUSTOM INSTRUCTIONS OVERRIDE
If custom_instructions is provided, apply it to the regenerated question. It may change tone, add constraints, modify structure, request patterns. Follow it unless it would produce factual errors.

RULE 2 — PRESERVE MULTI-PART STRUCTURE
If the source has sub-parts (a,b,c…), keep same count and order. Each sub-part tests the same concept as its source counterpart. Do NOT collapse multi-part into single unless explicitly requested or question_type conversion requires it.

RULE 3 — DISTRACTOR QUALITY (for MCQ/SCQ)
Every wrong option must be:
- Plausible (a common student mistake leads to it)
- Dimensionally correct (same units as correct answer)
- Numerically distinct (no two options within rounding error)
- Non-trivially wrong (not obviously absurd)

RULE 4 — NUMERICAL HYGIENE
- Change ALL numeric values in the stem.
- Use clean, curriculum-appropriate numbers.
- Verify: plug your answer back into the stem to make sure it functions properly.
- State units explicitly in the answer.

RULE 5 — COUNT FIDELITY
Generate EXACTLY one question. The list inside the JSON output under the "regenerated" key must contain exactly 1 item.

RULE 6 — RIGOROUS REPHRASING (for levels 2 and 3 — NOT for level 1 "Numbers only", which keeps the wording verbatim)
- Do NOT just swap synonyms — restructure the sentence entirely.
- Change the physical scenario (different object, setup, real-world context).
- The rephrased version must feel like a different author wrote it.
- Maintain an academic, rigorous, and precise tone.

RULE 7 — TYPE MATCHING
- If question_type is NOT specified: maintain the EXACT question type of the source.
- If question_type IS specified: convert to that type while preserving the concept.
- Match cognitive load — a 3-minute question stays a 3-minute question.

RULE 8 — STRUCTURE MIRRORING (match the source's shape, no more, no less)
The regenerated question MUST mirror the source's structure exactly. Do NOT add fields the source doesn't have; do NOT drop fields the source does have.

  • SOLUTION (always generated — exception to mirroring): ALWAYS include a solution for the regenerated question, even when the source had NO printed solution (e.g. most exercise questions).
    ORDER (MANDATORY — answer first): the solution string MUST start with a line in the exact form "Answer: <final answer>" giving the final result FIRST, then a blank line, then the justification. NEVER place the answer only at the end. Example:
        Answer: x = 3 and x = 4

        By factorisation, x² − 7x + 12 = (x − 3)(x − 4) = 0, so x = 3 or x = 4.
    Keep the justification CONCISE, formula-driven, and understandable: the governing formula(s)/method and only the key intermediate results. Do NOT show minor or trivial calculations (routine arithmetic, obvious substitutions, line-by-line algebra). If — and ONLY if — the solution genuinely needs a diagram to be understood, include that diagram as LaTeX code (TikZ/pgfplots/chemfig/circuitikz), the same format used for question diagrams. Solutions are ALWAYS produced and never omitted.
  • ANSWER KEY: if the source shows an inline answer, show the equivalent answer in the same inline style. If the source has no inline answer, leave the inline answer empty — do NOT fabricate one. (The final answer also appears at the START of the concise solution above.)
  • OPTIONS / MCQ FORMAT: if the source is an MCQ with printed options (A)/(B)/(C)/(D), the regenerated output must have the same number of options. If the source is subjective, do NOT add options.

═══════════════════════════════════════════
ANTI-HALLUCINATION GATE — 7 checks
═══════════════════════════════════════════
Before emitting the question, verify ALL seven:
(1) CONSISTENCY — all numbers in stem, options, answer, solution are mutually consistent
(2) CORRECTNESS — stated answer actually solves the stated stem
(3) UNITS — correct, consistent, curriculum-appropriate
(4) TYPE MATCH — output matches required question_type
(5) LABELS — stated concept and topic actually describe the question you wrote
(6) COMPLETENESS — all source sub-parts present, nothing out-of-scope introduced
(7) SOLUTION — begins with the final answer, then a concise formula-driven justification using the correct method/formula (no minor/trivial steps shown), internally consistent with the stem; any diagram it needs is in LaTeX format
