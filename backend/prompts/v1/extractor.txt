PROSE-AS-EQUATION GUARD — DO NOT misclassify English-words sentences as equations:
An English-words sentence with an `=` sign is PROSE, NOT an equation. Examples:
  - "Density = mass / volume"  → emit as `body`, NOT `equation`
  - "Speed = distance / time"  → emit as `body`, NOT `equation`
  - "The mass of a proton = 1.0078 u"  → emit as `body` (value statement)
  - "Binding energy of a nucleus = Δm × 931.5 MeV"  → emit as `body`
Only emit `equation` when the printed line is typeset as a standalone mathematical expression using symbols/variables set apart from prose (e.g., "ρ = m/V", "E = mc²", "x² + y² = r²"). If the line uses full English words for variables OR is a numerical value statement with units, it's `body` text.

CHEMISTRY FORMULAS — MANDATORY use `\ce{...}` (mhchem extension, loaded in KaTeX):
- Wrap EVERY chemistry formula in `\ce{...}` so subscripts/superscripts/bonds render correctly. NEVER write plain "H2O" in chemistry context — write `\ce{H2O}` (renders as H₂O).
    inorganic:     \ce{H2O}, \ce{NaCl}, \ce{H2SO4}, \ce{CO2}
    organic:       \ce{CH3-CH2-OH}, \ce{CH3-CHO}, \ce{C6H6}
    ions:          \ce{Fe^{2+}}, \ce{SO4^{2-}}, \ce{NH4+}
    isotopes:      \ce{^{14}_{6}C}, \ce{^{235}U}
    reactions:     \ce{2H2 + O2 -> 2H2O}, \ce{N2 + 3H2 <=> 2NH3}
    catalysts:     \ce{A ->[catalyst] B}, \ce{A ->[heat] B}
    states:        \ce{H2O(l) -> H2O(g)}
- For chemistry equations: put `\ce{...}` inside an `equation` block content, wrapped as `$$\ce{H2O}$$` or `\ce{H2O}` bare (pipeline normalizes wrapping).
- For inline chemistry in prose (`body`/`definition` content): wrap with `$\ce{...}$`. Example: `"The product is $\ce{H2SO4}$ (sulfuric acid)"`.
- Bond notation: use ASCII bonds `\ce{C-C}` (single), `\ce{C=C}` (double), `\ce{C#C}` (triple). The mhchem extension renders them as proper bond glyphs.
- NEVER use Unicode subscripts (H₂O), Unicode arrows (→), or Unicode bond chars (–, ≡). Always plain ASCII inside `\ce{...}`; mhchem handles rendering.

TABLE RULE — NON-NEGOTIABLE:

ANY grid / matrix / row-and-column structure printed on the page IS a table.
Emit it as type="table" — NEVER as type="body" (paragraph), NEVER as
type="key_point" (callout), NEVER concatenated into a single string with
newlines and pipes. Tables are STRUCTURED DATA, not prose.

If you see ANY of the following, it is a table — emit as type="table":
  - Two or more rows aligned in columns (with or without visible borders)
  - A comparison grid (e.g. "Element | Symbol | Atomic Number")
  - A properties table, periodic-table excerpt, formulae list with parallel columns
  - A data table with numeric values across columns
  - A row of column headers above aligned data rows
  - Side-by-side definitions / specifications (e.g. "Type A: ...  Type B: ...")
  - Reaction/process tables with reagents, conditions, products in columns

Fields (all required):
  - headers : array of column header strings (left-to-right). If the printed
              table has no header row, emit headers as [] and put the first
              row's content in rows[0].
  - rows    : array of rows. Each row is an array of cell strings, one per
              column. Cell count per row MUST equal headers.length (pad
              with "" if a cell is genuinely empty in the printed table).
  - caption : the printed table title / caption if present, else "".
