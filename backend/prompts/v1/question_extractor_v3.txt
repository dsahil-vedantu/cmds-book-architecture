LAYOUT DETECTION — apply BEFORE you transcribe each page
═══════════════════════════════════════════════════════════════════════════════

Look at the page geometry first. Determine column layout:

- SINGLE-COLUMN: full-width text flowing top-to-bottom. DEFAULT case for most
  NCERT textbooks. → Use STANDARD reading order. Ignore the MULTI-COLUMN
  block below; it does not apply.
- TWO-COLUMN: left column + right column with visible gutter. Common in
  MHT-CET / JEE / NEET prep books.
- THREE-COLUMN / FOUR-COLUMN: rare — three or four vertical streams side-by-side.
  Same multi-column reading rules apply: read each column FULLY top-to-bottom,
  left-to-right, before moving to the next page.

For SINGLE-COLUMN pages: continue with the standard rules below. No change
from existing behavior.

For MULTI-COLUMN pages (2 or 3 columns), apply these ADDITIONAL rules ON
TOP OF the standard rules:

──── MCQ ATOMIC-UNIT RULE ────
- A question = stem + (A)(B)(C)(D) options + (optional answer/solution).
- An atomic unit MUST NOT be split across your output. Stitch it together
  even when it physically wraps column boundaries.
- If a stem appears in column 1 bottom and its options continue at column 2
  top, they belong to ONE question — combine them in your output as a
  single `raw_text` value.
- If the stem + (A) + (B) are in column 1 and (C) + (D) are at column 2
  top, they STILL belong to ONE question — emit all four options in one
  `raw_text`.

──── OPTION ORDER RULE — critical ────
- (A), (B), (C), (D) MUST appear in ALPHABETICAL ORDER in your final
  `raw_text` output.
- If you encounter the printed sequence (A) then (C) without (B) between
  them, you've crossed a column boundary mid-question. STOP. Find (B) at
  the same vertical level in the OTHER column. Stitch (B) into the correct
  position. Then continue with (C) and (D).
- Verify before emitting: the option letters in `raw_text` should read
  A then B then C then D (or 1, 2, 3, 4) — alphabetically/numerically
  ascending with no gaps.
- If you cannot find (B) anywhere on the page (or (C), or (D)), emit the
  question with the options you DO have and flag the missing letter by
  leaving its position empty. NEVER reorder (A) and (C) to look right
  without finding (B).

──── MATH BRACE INTEGRITY across column wraps ────
- Math expressions like `\int_0^{π/2}`, `\frac{a}{b}`, `e^{2x}` must have
  every `{` matched with a `}`.
- Before emitting, count `{` and `}` in each `raw_text`. They MUST be
  equal in count.
- If you see `\int_0^{` with no closing `}` in column 1, look at column 2
  for the closing `}` — it's almost certainly the continuation of the
  same expression.
- NEVER emit a question whose `raw_text` has unmatched braces.

──── PRE-EMIT CHECKLIST per question (multi-column only) ────
Before adding a question to `extracted`, verify:
  ☐ Stem present and complete (ends with `?`, `=`, `is`, `are`, etc.)
  ☐ Options (A)(B)(C)(D) present and in ALPHABETICAL order
  ☐ All `{` and `}` braces balanced
  ☐ No options missing from the alphabetical sequence
If ANY check fails: re-read both columns, re-stitch the question, then
re-verify. If still failing after re-stitch, emit with the issue rather
than fabricate — the post-extraction validator will flag and retry.

──── SECTION BOUNDARY — critical for shared pages (multi-column only) ────
The user prompt for a multi-column book may include a line:

    BOUNDARY: This section starts on its first page AFTER the printed
    heading "<HEADING TEXT>". Any numbered items printed on that page
    BEFORE that heading belong to the previous section — do NOT include
    them, even if their question numbers look like "1.", "2.", etc.
    Start counting / extracting only from the first item printed AFTER
    the section's start heading.

When that line is present, you MUST:

1. On the FIRST page of the PDF slice, locate the printed heading text
   verbatim (case-insensitive, but the text otherwise must match).
2. Treat every printed item BEFORE that heading on that page as content
   of the previous section — IGNORE them entirely. Do not extract them.
   Do not include their question numbers in `identified_total`.
3. Start extraction with the FIRST item printed AFTER the heading.
4. Items on subsequent pages are part of this section as normal (until
   the STOP heading, if any).

Why this matters: in dense MCQ books (MHT-CET, JEE, NEET prep), one
printed page often carries the tail of section X in its first column
AND the beginning of section X+1 below a printed sub-heading. Without
the boundary anchor, the extractor mis-counts the tail of X as items
of X+1 (resetting the numbering at 1, 2, 3…) and runs out of capacity
before reaching the real items of X+1 further down the page. Result:
duplicate "Q1" / "Q2" rows in two sections + ~half the questions of
the second section silently dropped.

The boundary anchor is a HARD instruction. If you cannot find the
heading text on the first page, return `identified_total: 0` and add a
note in the `notes` field — do NOT extract content from a page where
you cannot locate the section's start.

WHAT COUNTS AS A QUESTION (INCLUDE)
═══════════════════════════════════════════════════════════════════════════════
A question-like item is anything the textbook is EXPLICITLY asking the reader
to answer, solve, work through, or prove. The item must be ANCHORED to a
visible printed marker — a number, label, or imperative verb on the page.
Specifically include:

  • Numbered exercises  ("1.", "Q.3", "Question 5", "Exercise 8.2")
  • Multiple-choice questions with their option lists (A/B/C/D)
  • Short-answer / long-answer prompts under "Short Answer" / "Long Answer"
  • Worked examples that pose a problem ("Example 4: Find the value of …",
    "Worked Example 5", "Solved Example 9.2")
  • Numbered Problems / Practice Problems printed as standalone numbered items
  • Numerical / derivation / proof problems
  • Review questions at the end of the section
  • MATCH-THE-COLUMN / MATCH-THE-FOLLOWING — emit as ONE single question.
    Even when the page numbers the row pairs (Q.6, Q.7, Q.8, Q.9, Q.10 inside
    the same Column-A/Column-B table), the WHOLE matching exercise is one
    question. Transcribe BOTH columns verbatim as a Markdown table inside
    raw_text so the column layout is preserved. Include every numbered row,
    every lettered option, and the instruction line ("Match the entries…").
    Example raw_text:
      "Match the entries in Column A with appropriate ones from Column B.
       | # | Column A | Column B |
       | - | -------- | -------- |
       | 6 | Coolidge tube | (a) present in all natural radioactive series |
       | 7 | α decay       | (b) isobars |
       | 8 | …             | (c) … |"
    Do NOT split the numbered rows into separate Question items. The block
    is ONE Q. `kind` = "other" (or "mcq" if explicit option choices follow).
    If the page also lists multiple-choice option codes ((A) 1-a, 2-b, 3-c…
    style) at the bottom selecting a permutation, include those verbatim
    inside the same raw_text.
  • ASSERTION–REASON items — each numbered Assertion+Reason+options block
    is ITS OWN item. If the page prints "Directions for questions N to M:"
    followed by the standard four-option rubric (A: A and R both true and
    R explains A; B: …; C: …; D: …), copy that full rubric into EACH
    numbered item's raw_text (do not abbreviate to "refer above"), then
    append the specific Assertion: / Reason: text for that numbered item.
    Emit one Question per printed number. Do NOT collapse Q.16, Q.17, Q.18,
    Q.19 into a single item just because they share the directions block.
  • COMPREHENSION / PASSAGE-BASED questions — when a printed passage or
    "Directions for questions N to M" preamble introduces several numbered
    questions, each numbered question is its OWN item. Copy the passage /
    directions text VERBATIM into EVERY numbered item's raw_text (yes, all
    of them — duplication is correct and required). Do NOT write
    "(refer to passage above)" — duplicate the full preamble each time so
    the question stands alone. Emit one Question row per printed number;
    if the page shows Q.11–Q.15, you MUST emit exactly 5 items.
  • LONG / ESSAY questions that span multiple lines or sub-parts — a single
    numbered essay prompt with parts (a)(b)(c) is ONE item; transcribe ALL
    sub-parts. Don't drop sub-parts because the question is long.

If you cannot point to a literal printed marker (a number, an "Example N", an
"Exercise N.M", a "Q.N", an explicit "Problem N" / "Practice Problem N", or
an imperative verb like "Find / Prove / Show / Calculate / Determine /
Evaluate / Derive" anchored to a numbered/labelled prompt), DO NOT EMIT IT.

RHETORICAL PROSE QUESTIONS ARE NOT QUESTIONS.
Theory paragraphs frequently contain question-shaped sentences for pedagogy:
"What is a function?" / "Why does this happen?" / "Can we always solve this?"
These are explanatory devices, NOT questions to extract. They appear in body
paragraphs, NOT under a numbered/labelled prompt. Treat them as theory.

WHAT IS NOT A QUESTION (EXCLUDE — DO NOT EMIT)
═══════════════════════════════════════════════════════════════════════════════
The single most common error is treating non-questions as questions.
EXCLUDE the following entirely — even if the page contains them:

  ✗ Crosswords, word-search puzzles, jumbles
  ✗ Fill-in-the-blank decorative panels with no question prompt
  ✗ "Hint" boxes / "Tip" boxes / "Note" boxes / "Remember" boxes
  ✗ "Did You Know" / "Fun Fact" / "Trivia" callouts
  ✗ Learning objectives ("After this section you will be able to …")
  ✗ Section summaries / key-points lists / bullet recaps
  ✗ Standalone answer keys printed at the end of the chapter
    (an answer printed inline with its question IS captured — see SOLUTION rule)
  ✗ Glossary / vocabulary lists
  ✗ Headings / sub-headings / running titles
  ✗ Page numbers, running headers, footers, publisher info, copyright lines
  ✗ Figure captions on their own (a caption is not a question)
  ✗ ANYTHING printed inside a styled box labelled Illustration / Progress
    Check / Activity / Try It / Quick Check / Test Yourself / Self-Check /
    Check Your Understanding — these are theory aids extracted by the theory
    pipeline. Even if they contain numbered prompts or question marks,
    return ZERO questions for content inside these boxes.
  ✗ ANYTHING printed inside a callout labelled Note / Remember / DO YOU KNOW /
    Thinking Corner / Key Point / Recall — these are inline theory callouts,
    NOT questions. Do not extract content inside them.
  ✗ Definitions, formulas, statements of laws, derivations presented as theory

If you are uncertain whether something is a question, ask yourself:
  "Is the textbook DIRECTING the reader, with a printed marker or imperative
   verb, to find / solve / state / prove / choose / calculate / derive / show /
   determine / evaluate / explain something?"
If the answer is no, EXCLUDE it. When in doubt, EXCLUDE.

MULTI-COLUMN PAGES — READ EVERY COLUMN:
Question banks frequently use 2, 3, or 4 columns per page. Read each column
FULLY top-to-bottom before moving to the next column, in left-to-right order.
Example: in a 2-column page, questions 1-15 may fill the LEFT column and 16-30
the RIGHT column — you MUST capture both columns. Never stop at the bottom of
the first column. Scan the whole page, all columns, before moving to the next page.

SHARED-CONTEXT COMPLETENESS — GENERIC RULE.
The single most common dropped-question failure is: a preamble (passage,
directions, case study, statement set, diagram caption) is followed by
multiple INDEPENDENTLY-NUMBERED items, and the extractor emits only the
first one. NEVER do this. The preamble is NOT a question — every numbered
item BELOW it is its own question.

Trigger phrases (treat ANY of these — and similar variants — as a shared-
context block; the list is illustrative, not exhaustive):
  • "Directions for questions N to M:" / "Directions (Q. N to M):"
  • "Read the following passage and answer the questions that follow"
  • "Based on the above, answer questions N to M"
  • "Study the following diagram/graph/table and answer"
  • "Case Study" / "Case-based questions" / "Source-based questions"
  • "Consider the following statements" / "Given below are two statements"
  • "Read the following information carefully"
  • "Comprehension" / "Passage I" / "Paragraph for Question Nos. N to M"
  • "Statement-I … Statement-II …" headers followed by numbered sub-items
  • Assertion-Reason rubrics shared across a numbered range
  • Any preamble paragraph followed by ≥2 numbered/lettered question items
    that clearly reference back to it (even without an explicit "Directions"
    keyword — recognise the pattern by the numbered items below)
  • Diagram / figure with a caption followed by multiple numbered Qs that
    depend on it
  • Numbered Roman / Arabic sub-items inside a master question that each
    carry their OWN question number (1, 2, 3 …) — these are separate Qs,
    NOT (a)(b)(c) sub-parts (see distinction below)

Procedure for every shared-context block you detect:
  1. Find the printed range. If the page prints "N to M", use that. Otherwise
     count the independently-numbered items BELOW the preamble that
     reference it. Set TARGET = number of numbered items.
  2. Emit EXACTLY TARGET separate Question rows — one per printed number.
     Never fewer. If TARGET is 5, you MUST output 5 rows. If TARGET is 19,
     you MUST output 19 rows.
  3. Copy the FULL preamble VERBATIM into each row's raw_text (yes, all of
     them — duplication across rows is correct and required), then append
     that row's specific question text (Assertion+Reason pair, comprehension
     question, statement-evaluation prompt, case-study sub-question, etc.).
     Do NOT abbreviate the preamble to "(refer to passage above)" — copy it
     in full so each row stands alone.
  4. Self-check before emitting JSON: for every shared-context block on the
     pages, count rows in `extracted` whose printed numbers fall in that
     block's range. If the count is less than TARGET, you have dropped
     sub-items — go BACK to the pages and find them. Repeat until counts
     match. Empty `extracted` for a shared-context block on the page is
     ALWAYS wrong unless the block truly belongs to a different section.

DISTINCTION — sub-parts vs separate questions:
  • (a)(b)(c) lettered sub-parts under ONE numbered question (e.g. "5. (a)
    Find x  (b) Find y  (c) Find z") → ONE Question item; transcribe all
    sub-parts inside its raw_text.
  • Independently numbered items 1, 2, 3 … each with their own printed
    question number, even sharing a preamble → SEPARATE Question items
    (one per number). The distinction is: does the page assign a new
    question NUMBER to each item? If yes → separate. If only letters
    (a/b/c) → one item with sub-parts.

EXCEPTION (Match-the-X tables only): a single "Match the columns" /
"Match the following" exercise stays as ONE Question even when its rows
carry numbers (Q.6, Q.7 …) — those numbers label the rows of one matching
exercise, not separate questions. Render the columns as a Markdown table
inside raw_text. This is the ONLY exception to the shared-context rule.

When in doubt between "one question with sub-parts" and "shared-context
block with N questions": if each item carries its own printed question
NUMBER, it is a separate question. Letter sub-parts (a)(b)(c) are NOT
question numbers.

If you finished reading and found zero question-like items, return:
  {"identified_total": 0, "extracted": [], ...}
That is a complete and correct answer for a pure-theory section.

HARD RULES — ANTI-HALLUCINATION
═══════════════════════════════════════════════════════════════════════════════
1. VERBATIM ONLY. Copy exact text including punctuation and numbering.
2. NO PARAPHRASE, NO SUMMARY, NO INFERENCE.
3. SUB-PARTS (a)(b)(c) belong to ONE item with the full text of every sub-part —
   UNLESS each sub-part carries an independent question number, in which case
   each is its own item.
4. MCQ OPTIONS — include every option choice verbatim in raw_text.
   NEVER add options to a question that does not visibly print them. If only a
   stem is printed with no options, kind is NOT "mcq" — it is "exercise" or
   "other", and `has_options: false`.
5. ALL MATH IN LATEX inside delimiters $...$ (inline) or $$...$$ (display).
   Replace unicode math characters: × → \times, ÷ → \div, ± → \pm, ≤ → \leq,
   ≥ → \geq, ≠ → \neq, ≈ → \approx, ∞ → \infty, ° → ^{\circ}, π → \pi,
   θ → \theta, √x → \sqrt{x}, a² → a^{2}, a₁ → a_{1}, ½ → \frac{1}{2}, etc.
   Numbers joined by operators are math: "3 + 4 = 7" → "$3 + 4 = 7$".
   Do NOT simplify expressions; transcribe as printed.
   CHEMISTRY: wrap every chemistry formula in \ce{...} inside math delimiters —
   inline: $\ce{H2SO4}$, $\ce{Fe^{2+}}$, $\ce{CO2}$. Reactions: $\ce{2H2 + O2 -> 2H2O}$.
   Use ASCII inside \ce{} (H2O not H₂O, -> not →). Never leave a formula as plain text.
6. FIGURES — insert {{fig: <printed label> — <caption>}} at the exact position
   where the figure appears in the question text. Use the printed label
   verbatim ("Fig. 4.5"). If unlabelled: "(unlabelled diagram)".
   NEVER insert a figure placeholder if no figure is visibly attached to the
   question on the page. Do not "expect" a figure that isn't there.
7. SOLUTIONS — capture in `solution` ONLY the solution text that is literally
   printed adjacent to the question on the page (worked-example solutions,
   inline answer keys). Apply the same LaTeX and figure rules.
   NEVER compute, complete, simplify, derive, or supply a missing solution.
   NEVER use your training knowledge to fill in steps the page does not show.
   If no solution is printed, `solution: ""` and `has_solution: false`.
   If the printed solution stops mid-step or is partially illegible,
   transcribe what is readable and stop. Do not finish it for the textbook.
   WORKED EXAMPLES & SOLVED PROBLEMS: every EXAMPLE / Solved-example
   item has a COMPLETE printed solution immediately after the problem
   statement. You MUST capture the full solution (all steps, equations, final
   answer) in the `solution` field. Never extract the problem without its
   printed solution. If — and ONLY if — no solution is printed, leave
   `solution` empty (never fabricate).
8. WORKED EXAMPLE STRUCTURE — when both a problem statement and a solution are
   printed for the same example, put ONLY the problem statement (the part the
   reader is asked to solve) in `raw_text`, and the printed solution in
   `solution`. Do not merge them. Do not invent a problem statement when only
   a derivation is printed without an explicit "find / show / prove" prompt —
   that is theory, not a question.
9. RAW_TEXT LENGTH SANITY — a single question's raw_text rarely exceeds the
   length of one column of one page. If your raw_text grows much longer, you
   are probably scooping up surrounding theory or the next question. Re-read
   and tighten the boundary. Each question is ONE printed item.
10. NEVER FABRICATE. If unsure → omit. If unclear → "[illegible]". An empty
    `extracted` list for a pure-theory section is correct.

11. INTERNAL CONSISTENCY (mandatory):
    - has_options=true → the `raw_text` field MUST contain the printed
      option markers (e.g. "(A) ... (B) ... (C) ... (D) ...") verbatim.
    - has_solution=true → the `solution` field MUST be a non-empty string
      containing the verbatim transcribed solution text.
    - If you cannot transcribe the solution verbatim, set has_solution=false
      and leave `solution` null — DO NOT set has_solution=true with an
      empty solution field.
    - If you cannot transcribe all options verbatim, set has_options=false
      and explain in `raw_text` via [UNCLEAR: ...] markers.
